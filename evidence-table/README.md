## 证据表填写规则

1. **每个岗位 1 张表**，不要合并多张
2. **第三列是关键**——直接写"我已有的 repo + 该 repo 的状态"
3. **没有就写"❌ 无"**——这就是缺口，不要骗自己
4. **跨岗位交集 = 30 天作品目标**——找最大缺口
5. **每两周迭代一次**——W2 末、W4 末、W6 末、W8 末

---

## 三列拆解的核心

```
第一列：必须会的技术     →  从 JD 提取
第二列：必须交付的结果   →  从 JD 提取
第三列：我现在能拿出什么 →  从自己 repo + 经历提取
```

**最大缺口 = 出现频次高 + 你最缺 = 30 天作品目标**
