> **方法论基础**：岗位证据法 —— 三个问题 + 四步实操
> 1. 我要进入哪一类岗位？
> 2. 这个岗位要我交付什么结果？
> 3. 我现在有什么作品或经历能证明我做得到？

## 四、HexoAIAgent RAG 升级版架构

```
用户提问
   ↓
查询解析（意图识别）
   ↓
Hexo 博客文档检索（Markdown 文件 → 切片 → Embedding → 向量库）
   ↓
RAG 检索 Top-K
   ↓
LLM 生成答案 + 文章链接
   ↓
返回结构化响应
```

**6 步 checklist**（视频里 Agent 工程）：
1. **定义边界**：能做什么（基于博客知识库的回答）/ 不能做什么（编造、未读文章）
2. **整理上下文**：博客目录解析 + 切片策略 + 标注来源
3. **设计工具调用**：搜索工具 + 重新检索工具 + 反馈工具
4. **人工确认**：答案来源不确定时请求确认
5. **评测**：典型 3 + 异常 3 + 边界 3 = 9 条样例
6. **观测与复盘**：记录 query / 检索结果 / LLM 输出 / 用户反馈

## 五、避坑清单（视频反例 + 自己的）

- ❌ 不要 fork mall 当作品 —— 面试官一问就露馅

- ❌ 不要一个人闷头做 —— 每周找 Mavis 或同学 review 30 分钟
- ❌ 不要追求完美架构 —— B 同学宿舍系统就够了
- ❌ 不要卷算法岗（除非研究生）
